# 本地音乐资源部署指南

本文档说明 Layer3 推荐层与本地音乐管理模块的关系，以及如何部署和使用。

---

## 架构概览

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    本地音乐管理模块 (LocalMusicModule)                    │
├─────────────────────────────────────────────────────────────────────────┤
│ 职责：                                                                   │
│ 1. 读取预先生成的 index.db 数据库文件                                    │
│ 2. 将 index.db 数据传输至 Layer3 推荐层                                  │
│ 3. 接收并处理 Layer3 生成的音乐歌单数据                                  │
│ 4. 负责播放本地音乐资源文件                                              │
│                                                                          │
│ 不负责：                                                                 │
│ - 场景分析、推荐逻辑（由 Layer3 负责）                                   │
│ - 歌单生成策略（由 Layer3 负责）                                         │
└─────────────────────────────────────────────────────────────────────────┘
                                    ↓↑ 数据交互
┌─────────────────────────────────────────────────────────────────────────┐
│                      Layer3 推荐层 (ContentEngine)                       │
├─────────────────────────────────────────────────────────────────────────┤
│ 职责：                                                                   │
│ 基于 Layer2 输入的描述信息（模板或大模型生成）与本地音乐管理模块传入的    │
│ index.db 数据，生成符合场景需求的音乐歌单                                │
│                                                                          │
│ 推荐策略（三层）：                                                       │
│ 1. 模板场景：SQL 查询直接匹配 → 快速启播                                 │
│ 2. 非模板场景：调用大模型智能选取 → 个性化歌单                           │
│ 3. 兜底场景：预设固定音乐清单 → 备选方案                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

## Layer3 三层推荐策略

### 策略流程图

```
curatePlaylist(sceneType, hints)
    │
    ├─ 判断 sceneType 是否为模板场景
    │   TEMPLATE_SCENES = [morning_commute, night_drive, road_trip, ...]
    │
    ├─ 策略1: 模板场景 → SQL 直接匹配
    │   条件: isTemplateScene(sceneType) && localDb 可用
    │   执行: SELECT * FROM tracks WHERE scene_tags LIKE '%sceneType%'
    │   延迟: < 10ms
    │   source: 'sql'
    │   有结果 → 返回歌单 ✅
    │   无结果 → 继续 ↓
    │
    ├─ 策略2: 非模板场景 → LLM 智能选取
    │   条件: llmClient 可用
    │   执行: 调用大模型，传入 index.db 数据摘要
    │   延迟: 3-10s
    │   source: 'llm'
    │   有结果 → 返回歌单 ✅
    │   失败 → 继续 ↓
    │
    └─ 策略3: 兜底场景 → 预设固定清单
        条件: 以上都失败
        执行: 返回 FALLBACK_PLAYLIST（10首周杰伦歌曲）
        延迟: < 1ms
        source: 'fallback'
        返回歌单 ✅
```

## 数据流

```
Layer2 输出 (SceneDescriptor)
    │
    │  { scene_type: 'morning_commute', hints: { genres: ['pop'], energy: 0.6 } }
    │
    ▼
Layer3 推荐层
    │
    │  1. 判断 scene_type 是否为模板场景
    │  2. 从本地音乐模块获取 index.db 数据
    │  3. 执行推荐策略
    │  4. 输出歌单
    │
    ▼
输出格式:
{
  playlist: [
    { id, title, artist, genre, bpm, energy, filePath, ... }
  ],
  source: 'sql' | 'llm' | 'fallback',
  strategy: '模板场景：SQL 查询直接匹配，快速启播',
  scene_match: {
    type: 'template',
    scene: 'morning_commute',
    matchScore: 1.0,
    matchMethod: 'sql_query'
  }
}
    │
    ▼
本地音乐管理模块
    │
    │  1. 接收歌单数据
    │  2. 加载音乐文件
    │  3. ExoPlayer 播放
    │
    ▼
用户听到音乐 🎵
```

## 接口定义

### 本地音乐管理模块 → Layer3

```typescript
interface LocalMusicModule {
  // 获取数据库状态
  getStats(): { totalTracks: number, genres: string[] };
  
  // 执行 SQL 查询
  queryTracks(sql: string, params: any[]): Track[];
  
  // 检查是否就绪
  isReady(): boolean;
}
```

### Layer3 → 本地音乐管理模块

```typescript
interface ContentEngine {
  // 生成歌单
  curatePlaylist(
    hints: MusicHints,
    constraints: Constraints,
    sceneType: string | null
  ): PlaylistResult;
  
  // 设置本地音乐模块
  setLocalMusicModule(module: LocalMusicModule): void;
}
```

## 部署步骤

### 1. PC 端整理音乐

```bash
cd tools/music-organizer

# 使用 LLM 分析标签
python organize_music.py \
  --input /path/to/your/music \
  --output ./index.db \
  --llm \
  --api-key sk-xxx
```

### 2. 传输到车机

```bash
# 推送音乐文件
adb push /path/to/your/music/ /sdcard/Music/AiMusic/music/

# 推送索引数据库
adb push ./index.db /sdcard/Music/AiMusic/index.db
```

### 3. App 使用

```kotlin
// Android 端
LocalMusicModule.initialize()
val repository = LocalMusicModule.getRepository()

// 查询音乐
val tracks = repository?.queryTracks(hints)

// 播放
val player = MusicPlayer(context)
player.play(tracks)
```

## 常见问题

### Q: 什么情况走 SQL 匹配？

A: 当 `sceneType` 在 `TEMPLATE_SCENES` 列表中，且 `localDb` 可用时，直接通过 SQL 查询匹配音乐。

### Q: 什么情况走 LLM？

A: 当 `sceneType` 不在模板列表中，或者 SQL 查询无结果时，调用 LLM 智能选取。

### Q: LLM 如何使用 index.db 数据？

A: LLM 会收到 index.db 的摘要信息（总曲目数、流派列表、部分曲目示例），然后从中选取最适合的歌曲。

### Q: 兜底清单是什么？

A: 预设的 10 首周杰伦歌曲，覆盖多种情绪和能量级别，作为最后的备选方案。
