# 本地音乐资源部署指南

本文档说明如何将本地音乐资源整理、部署到车机，并在 App 中使用。

---

## 整体流程

```
PC 端整理 → 生成索引 → 传输到车机 → App 加载 → 检索播放
```

---

## 第一步：PC 端整理音乐

### 1. 准备音乐文件

将音乐文件按流派分类存放：

```
/path/to/your/music/
├── pop/
│   ├── 周杰伦 - 晴天.mp3
│   └── Taylor Swift - Shake It Off.mp3
├── rock/
│   └── Linkin Park - In The End.mp3
├── jazz/
├── classical/
└── ...
```

### 2. 运行整理工具

```bash
cd /Users/mi/Desktop/音乐推荐项目/tools/music-organizer

# 激活虚拟环境
source venv/bin/activate

# 运行工具
python organize_music.py \
  --input /path/to/your/music \
  --output ./music_package/index.db \
  --format all
```

### 3. 输出产物

```
music_package/
├── index.db          # SQLite 索引数据库
├── index.json        # JSON 格式索引 (可选)
└── index.csv         # CSV 格式索引 (可选)
```

---

## 第二步：传输到车机

### 方式一：ADB 推送（开发阶段）

```bash
# 推送音乐文件
adb push /path/to/your/music/ /sdcard/Music/AiMusic/music/

# 推送索引数据库
adb push ./music_package/index.db /sdcard/Music/AiMusic/

# 验证
adb shell ls -la /sdcard/Music/AiMusic/
```

### 方式二：USB/SD 卡（Demo 阶段）

1. 将音乐文件和 `index.db` 复制到 USB 或 SD 卡
2. 在车机上插入 USB/SD 卡
3. 使用文件管理器复制到 `/sdcard/Music/AiMusic/`

### 方式三：应用内下载（生产阶段）

1. 首次启动时检测音乐库
2. 如不存在，提示用户下载
3. 后台下载并解压到指定目录

## 第三步：App 使用

### Node.js 端

```javascript
const { ContentEngine } = require('./src/layers/effects/engines/content');

// 初始化（自动加载 /sdcard/Music/AiMusic/index.db）
const engine = new ContentEngine({
  localDbPath: '/sdcard/Music/AiMusic/index.db',
  apiKey: process.env.DASHSCOPE_API_KEY
});

// 检查本地数据库状态
console.log(engine.getLocalDbStats());

// 检索音乐
const result = await engine.curatePlaylist(
  { genres: ['pop'], energy_level: 0.6, tempo: 'fast' },
  {},
  'morning_commute'
);

console.log(result.source);  // 'local' | 'library' | 'llm' | 'mock'
console.log(result.playlist); // [{ title, artist, filePath, ... }]
```

### Android 端

```kotlin
// 初始化模块
LocalMusicModule.initialize()

// 获取 Repository
val repository = LocalMusicModule.getRepository()

// 按 MusicHints 查询
val hints = MusicHints(
    genres = listOf("pop", "rock"),
    tempo = "moderate"
)
val tracks = repository?.queryTracks(hints)

// 使用 ExoPlayer 播放
val player = MusicPlayer(context)
player.play(tracks)
```

## 数据库结构

### tracks 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| title | TEXT | 歌名 |
| title_pinyin | TEXT | 歌名拼音 |
| artist | TEXT | 歌手 |
| artist_pinyin | TEXT | 歌手拼音 |
| album | TEXT | 专辑 |
| genre | TEXT | 流派 |
| bpm | INTEGER | 节拍数 |
| energy | REAL | 能量值 (0-1) |
| valence | REAL | 情绪正向度 (0-1) |
| mood_tags | TEXT | 情绪标签 JSON |
| scene_tags | TEXT | 场景标签 JSON |
| duration_ms | INTEGER | 时长（毫秒） |
| file_path | TEXT | 文件路径 |
| format | TEXT | 格式 (mp3/flac/wav/m4a) |
| bitrate | INTEGER | 比特率 |

### tracks_fts 虚拟表

FTS5 全文检索表，支持快速搜索歌名、歌手、专辑等。

## 检索策略

ContentEngine 的四级策略：

| 优先级 | 来源 | 条件 | 延迟 |
|--------|------|------|------|
| 1 | 本地 SQLite | `localDb` 可用且有匹配 | **< 10ms** |
| 2 | 预设音乐库 | `sceneType` 在预设列表中 | < 5ms |
| 3 | LLM 生成 | 前两级无结果且 API 可用 | 3-10s |
| 4 | Mock 数据 | 兜底 | < 1ms |

## 常见问题

### Q: 音乐文件路径不正确怎么办？

A: 确保音乐文件的实际路径与 `index.db` 中的 `file_path` 一致。可以在整理时指定 `--base-path` 参数：

```bash
python organize_music.py \
  --input /path/to/music \
  --output ./index.db \
  --base-path /sdcard/Music/AiMusic/music
```

### Q: 如何更新音乐库？

A: 重新运行整理工具，生成新的 `index.db`，然后推送到车机覆盖原文件。

### Q: 支持哪些音频格式？

A: MP3、FLAC、WAV、M4A。Android 端使用 ExoPlayer 播放，支持大多数常见格式。

## 延迟对比

| 方案 | 延迟 |
|------|------|
| 本地 SQLite 检索 | **< 10ms** |
| LLM 生成歌单 | 3-10s |

本地检索比 LLM 快 **1000 倍**，用户体验显著提升。
